# Governance Plan Visualizer

## Overview

Transform a governance brief into a beautiful, self-contained interactive HTML dashboard branded for Credo AI. The output is a single `.html` file — no server, no dependencies to install, just open in a browser. Serves two audiences: executives who need a posture summary, and the AI/risk team who need to explore risks, compliance obligations, and controls in depth.

## What to produce

A single self-contained HTML file with these sections, in order:

1. **Header** — Credo AI logo, system name, deployment status badge, autonomy level, domain, jurisdiction
2. **Sticky navigation bar** — slim bar with jump links to each section
3. **Executive Overview** — system context card, risk posture tiles, top actions, compliance snapshot, gaps flag
4. **Risk View** — 5×5 risk matrix + sortable expandable risk table with control linkage
5. **Compliance View** — obligations grouped by source regulation, annotatable checklist
6. **Action View** — swim lane control cards showing mitigated risks and policy requirements
7. **Governance Gaps** — Critical/High risks with no associated controls
8. **Footer** — "Generated by Credo AI Governance Hub · [ISO date]" — read `./docs/credoai/org.md` (local) or `~/.claude/credoai/org.md` (global fallback); if either exists, append " · Prepared for [Organization Name]"

### `<title>` element

Pin the document title in `<head>` to exactly this format:

```html
<title>{System Name} — Governance Plan | Credo AI</title>
```

`{System Name}` is the title-cased system name from the governance brief (e.g. `RetailAssist`). The Governance Hub backend extracts this `<title>` to display in published-artifact listings, and `aigov-share` sends `artifactType: "plan"` alongside, so the title format must stay consistent for the listing UI to render correctly.

## Technical approach

Single self-contained HTML file: vanilla JS + inline CSS, no build step, no npm. Tailwind via CDN is fine for layout utilities, but **all color, type, spacing, radius, and shadow values must come from the bundled Credo AI design tokens** (see below) — do not substitute Tailwind defaults.

```html
<script src="https://cdn.tailwindcss.com"></script>
```

The risk matrix is a pure CSS grid — no charting library needed.

### Design tokens — inline at gen time

The Credo AI design system ships a single CSS variables file (`credo-design-tokens.css`) bundled as an asset in this skill. **Read it at generation time and inline its contents inside `<style>` tags in the `<head>`.** This gives the dashboard the full canonical token set (colors, type scale, spacing, radii, shadows) used across the Credo AI platform — and means every viz output stays in sync with the brand kit when it updates.

```python
import glob, os
patterns = [
    os.path.expanduser("~/.claude/plugins/cache/credoai-plugins/governance-intelligence-pro/*/skills/aigov-plan-viz/assets/credo-design-tokens.css"),
    os.path.expanduser("~/.claude/plugins/cache/*/governance-intelligence-pro/*/skills/aigov-plan-viz/assets/credo-design-tokens.css"),
]
tokens_css = ""
for pattern in patterns:
    matches = glob.glob(pattern)
    if matches:
        with open(matches[0]) as f:
            tokens_css = f.read()
        break
```

Embed in the document head:

```html
<style>
{{tokens_css}}
/* viz-specific overrides go below the canonical tokens */
</style>
```

If the asset is missing, fall back to a minimal inline `:root { --credo-purple:#7B04CA; --grey-900:#363644; ... }` block covering the values referenced by this skill — never leave the dashboard token-less.

### Typography — Inter for product, Instrument Sans for hero

The design system splits type by surface: **Inter** is the product face (used in the platform UI for density and readability at small sizes), **Instrument Sans** is the marketing/brand face. Dashboards are hybrid — they're presentational, but they carry dense data. The convention:

- **Body, tables, cards, pills, buttons, captions:** Inter (`var(--font-sans)`)
- **Hero system name in the dark header banner only:** Instrument Sans (`var(--font-marketing)`) — this is the one brand-marketing moment in the artifact

Both fonts are imported by `credo-design-tokens.css`. Apply globally:

```css
body { font-family: var(--font-sans); color: var(--grey-800); background: var(--ghost-white); }
.font-marketing { font-family: var(--font-marketing); }
```

**Product type ladder** (from `colors_and_type.css`, derived from the platform Figma):

| Token | Size | Weight | Use |
|-------|------|--------|-----|
| `--fs-display` | 40px | 700 / -0.02em | Hero system name |
| `--fs-h1` | 32px | 700 / -0.01em | Page-level titles |
| `--fs-h2` | 24px | 700 | Section headings |
| `--fs-h3` | 18px | 700 | Card titles |
| `--fs-h4` | 16px | 600 | Sub-headers, tile labels |
| `--fs-body-lg` | 15px | 400 | Default body |
| `--fs-body` | 14px | 400 | Table cells, dense body |
| `--fs-body-sm` | 13px | 500 / +0.012em | Button + input labels |
| `--fs-sm` | 12px | 400 | Hint, metadata |
| `--fs-overline` | 10px | 700 / +0.08em uppercase | Eyebrows, group labels |

Use weight (not color) for emphasis. **No italics** for decorative emphasis — bold instead. Body text colors are `--grey-800` on light surfaces and `--white` on dark.

### Design principles (from the brand guide and the platform Figma)

- **Less is more** — generous whitespace, don't clog the layout. The brand guide repeats this rule four times.
- **Strong text hierarchy via weight, not color** — use 400/500/600/700; never colorize body words. Highlights are bold, not violet.
- **Left-aligned text** — always, except in centered hero moments.
- **Sentence case in body and UI** — no ALL-CAPS for emphasis. The only uppercase usage is the eyebrow/overline (10px / 700 / +0.08em tracking).
- **Rounded corners (product scale)** — `--r-sm` (6px) for buttons, status pills, and inputs; `--r-md` (8px) for small cards and list items; `--r-lg` (12px) for primary content cards; `--r-xl` (16px) for hero panels. Reserve `--r-pill` (999px) for header badges only — the product surface uses 6px pills, not capsules.
- **Sharp corners on data tables** — the brand guide is explicit; don't round table cells.
- **Soft purple-tinted shadows** — never hard black drop-shadows. Use `--shadow-xs` / `--shadow-sm` for cards (the elevation default), `--shadow-md` for hover, `--shadow-lg` for floating menus and modals only.
- **Gradient for movement** — French Violet → Mimi Pink (`linear-gradient(90deg, var(--credo-purple), var(--mimi-pink))`) as a 2–3px accent line at the bottom of the header. Never on backgrounds, never on text.
- **No emoji, no decorative italics, no left-border-accent cards** — these are explicitly banned in the brand guide.

### Iconography — Feather Icons via CDN

The brand uses **Feather Icons** (1.5–2px stroke, square endpoints, non-rounded corners) — never emoji, never hand-rolled SVGs. Load via CDN and let `feather.replace()` swap `<i data-feather>` placeholders into inline SVGs:

```html
<script src="https://unpkg.com/feather-icons"></script>
<i data-feather="alert-triangle" style="width:16px;height:16px"></i>

<script>feather.replace({ 'stroke-width': 2 });</script>
```

Re-call `feather.replace()` after dynamic content insertion. Sizes: 12 / 14 / 16 / 20 / 24 px (no fractional). Color icons by setting `color:` on the parent — `feather.replace()` produces `stroke="currentColor"`. Use `--credo-purple` for active-state icons; `--grey-700` for inline meta icons; white on dark surfaces.

## Data structure

Embed as a JS constant at the top of the `<script>` block:

```javascript
const DATA = {
  system: { name: "...", status: "...", autonomy: "...", domain: "...", jurisdiction: "..." },
  risks: [{ name, type, severity, likelihood, score, tier, rationale }],
  controls: {
    doNow:    [{ label, mitigates: [riskName, ...], policyRequirement }],
    doNext:   [{ label, mitigates: [riskName, ...] }],
    quickWins:[{ label, mitigates: [riskName, ...], note }]
  },
  compliance: [{ text, source }],
  gaps: ["risk name", ...]
};
```

Derive the control→risk reverse lookup at runtime (no new data required):

```javascript
const riskControls = {};
[...DATA.controls.doNow, ...DATA.controls.doNext, ...DATA.controls.quickWins]
  .forEach(c => (c.mitigates || []).forEach(r => {
    riskControls[r] = riskControls[r] || [];
    riskControls[r].push({ label: c.label, lane: detectLane(c) });
  }));

function detectLane(c) {
  if (DATA.controls.doNow.some(x => x.label === c.label))   return 'doNow';
  if (DATA.controls.doNext.some(x => x.label === c.label))  return 'doNext';
  return 'quickWins';
}
```

Lane is used to color "Addressed by" chips in the Risk View:
- `doNow` → Critical tier colors (`#260838` bg, white text)
- `doNext` → High tier colors (`#7B04CA` bg, white text)
- `quickWins` → Medium tier colors (`#C9B6FF` bg, `#161616` text)

## Section 1: Executive Overview

**System context card** — white card (`border: 1px solid var(--grey-200)`, `border-radius: var(--r-lg)`, `box-shadow: var(--shadow-sm)`). System name in `--fs-h1` / 700. One-line description confirming what is being assessed (`--fs-body-lg` / 400 / `--grey-700`). Below: pill badges for deployment status, autonomy, domain, jurisdiction — use the **product Pill recipe** (`primary` tone: `bg #F1EBFF / fg #41006B`, `border-radius: var(--r-sm)` 6px, Inter 11 / 600 / +0.012em). Each pill optionally leads with a 12px Feather icon (`flag`, `cpu`, `award`, `globe`).

**Risk posture strip** — four large-number tiles (Critical / High / Medium / Low counts), color-coded by tier. Same as the current summary stats strip.

**Top actions** — heading "Immediate Actions", then the first three `DATA.controls.doNow` items as a numbered list (label text only, no expansion). Below the list, a small text link "See all actions →" that scrolls to the Action View section.

**Compliance snapshot** — heading "Compliance Obligations". Group `DATA.compliance` by `source` and show each source with its count: e.g., "EU AI Act · 4 obligations". If `source` is empty, use "General". Each source on its own line as a small stat. Text link "See full checklist →" scrolls to Compliance View.

**Gaps flag** — if `DATA.gaps.length > 0`, render a callout card: `background: #260838; color: #FFFFFF; border-radius: var(--r-lg); padding: 16px; display: flex; gap: 12px; align-items: center`. Lead with a Feather `alert-triangle` icon at 20px in `--mimi-pink`. Text: "[N] Critical/High risk[s] have no identified controls." (No emoji — the Feather icon is the affordance.) Otherwise omit this block entirely.

## Section 2: Risk View

**Risk matrix** — 5-column × 5-row CSS grid. Severity on the X axis (1–5, left to right). Likelihood on the Y axis (5–1, high at top). Each risk is a small pill badge placed in its (severity, likelihood) cell — allow text wrapping (`white-space: normal; word-break: break-word`). Multiple risks in the same cell stack vertically. Cell backgrounds use the muted tier colors from the table above.

**Risk table** — sortable columns: Risk, Type, Severity, Likelihood, Score, Tier. Click a column header to sort ascending; click again for descending. Default sort: Score descending.

Each row is expandable (click anywhere on the row). Expanded state shows two sub-rows:

1. **Rationale** — the full `rationale` string, left-aligned, `color: #161616`, `font-size: 0.875rem`.

2. **Addressed by** — label "Addressed by:" followed by chips. Source from `riskControls[risk.name]`:
   - One chip per entry, text = `entry.label`, colored by `entry.lane` (see lane colors in Data structure section above).
   - Clicking a chip: smooth-scroll to the matching control card in the Action View and briefly highlight it (add a `box-shadow: 0 0 0 3px #7B04CA` for 1.2 seconds, then remove).
   - If `riskControls[risk.name]` is empty or undefined: single chip reading "No controls identified", styled with Low tier colors (`#E7C9FA` bg, `#161616` text).

## Section 3: Compliance View

One card per unique `source` value in `DATA.compliance`. Sort cards alphabetically by source name. Card header: lead with a Feather `book-open` icon at 16px (`--credo-purple`), source name in `--fs-h3` / 700, plus obligation count using the **product Pill `primary` tone** (`bg #F1EBFF / fg #41006B`, `border-radius: var(--r-sm)`).

Items with an empty or missing `source` go into a card labelled "General Obligations", sorted last.

Within each card, render each obligation as:
- `<input type="checkbox">` — ephemeral interactive state (no persistence)
- Obligation text, `color: #161616`

No control or risk cross-references in this section — current data does not include them.

## Section 4: Action View

Three equal-width swim lane columns side by side (use CSS grid: `grid-template-columns: 1fr 1fr 1fr`). On narrow viewports, stack vertically.

Column headers:
- "Do Now" — `background: #260838; color: #FFFFFF`
- "Do Next" — `background: #7B04CA; color: #FFFFFF`
- "Quick Wins" — `background: #C9B6FF; color: #161616`

All headers: `border-radius: var(--r-md) var(--r-md) 0 0; padding: 12px 16px; font-family: var(--font-sans); font-weight: 700; font-size: var(--fs-h4)`. Lead each header with a Feather icon at 16px: `zap` (Do Now), `clock` (Do Next), `target` (Quick Wins).

Each control card (`background: #FFFFFF; border: 1px solid var(--grey-200); border-radius: var(--r-lg); box-shadow: var(--shadow-xs); margin-bottom: 12px; padding: 16px`):

1. Control `label` — bold, `color: #161616`

2. Mitigates chips — label "Mitigates:" in small gray text, then one chip per name in `c.mitigates`. Color each chip by the matching risk's tier: look up `DATA.risks.find(r => r.name === riskName)?.tier` and apply tier colors. Clicking a chip: smooth-scroll to the matching risk row in the Risk View table and expand it.

3. For `doNow` controls only: if `policyRequirement` is present, render a pill below the risk chips using the **product Pill `ghost` tone with a brand-purple border**: `bg #FFFFFF / fg #7B04CA / 1px border #7B04CA / border-radius: var(--r-sm) / padding: 2px 8px / Inter 11 / 600 / +0.012em`. Prefix text: "Policy: " + `policyRequirement` value.

4. For `quickWins` controls only: if `note` is present, show an expand chevron (▾). Clicking the card toggles a `note` sub-row (`color: #555; font-size: 0.875rem; padding-top: 8px; border-top: 1px solid #E7C9FA; margin-top: 8px`). `doNow` and `doNext` cards have no expand affordance.

## Section 5: Governance Gaps

Render only risks that appear in `DATA.gaps`. For each gap, look up the full risk entry in `DATA.risks` by name.

Each gap item is a card (`background: var(--ghost-white); border: 1px solid var(--grey-200); border-radius: var(--r-lg); padding: 16px`). Lead with a Feather `alert-triangle` icon at 16px in the gap's tier color:
- Risk name bold
- Tier badge (use tier colors)
- Rationale text (`color: #161616; font-size: 0.875rem`) — from `DATA.risks` entry. If the risk name is not found in `DATA.risks`, show name and tier badge only.

If `DATA.gaps` is empty, render a single card with text "No governance gaps identified." in `color: #555`.

## Parsing the governance brief

The brief arrives as Markdown. Extract:

- **Header block**: system name, deployment status, autonomy, domain, jurisdiction
- **Risk table rows**: name, type, severity (int), likelihood (int), score (int), tier, rationale
- **Control roadmap items**: for each swim lane, parse lines of the form `- [label] — mitigates: [Risk A], [Risk B]; [optional note or policy text]`.
  - For Do Now lines: `- [label] — satisfies [policy text]; mitigates: [risks]` — extract `label`, `policyRequirement`, and `mitigates` array.
  - For Do Next lines: `mitigates: [risks]` — extract `mitigates` array.
  - For Quick Wins lines: `mitigates: [risks]; [note text]` — extract `mitigates` array and `note`.
- **Compliance obligations**: each bullet as a string; extract `source` from parentheses or after a dash at the end of the line (e.g., "You must document model training data *(EU AI Act Art. 10)*" → source "EU AI Act Art. 10"). If no source marker, leave `source` empty.
- **Governance gaps**: risk names listed under the Governance Gaps heading
